# RemoteRewriteApp

Technitium DNS App for remote-managed rewrite overrides.

What it does:
- fetches one or more remote rewrite sources
- refreshes it on a timer inside Technitium
- matches `suffix`, `glob`, and `regex` rules
- returns `A`, `AAAA`, or `CNAME` answers

Supported source formats:
- `adguard-filter`
  - direct drop-in of existing AdGuard-style `dns.txt`
  - the app extracts only `$dnsrewrite=` rules
- `rewrite-rules-json`
  - optimized manifest generated by `private content repo`

Recommended use:
- keep block/allow lists in native Technitium block list URLs
- use this app only for rewrite overrides that need AdGuard-like behavior

Repository role:
- this repo is only the Technitium app
- the filter/generator repo is separate: `private content repo`
- `private content repo` publishes `rewrite-rules.json`
- this app can fetch either the existing AdGuard-style `dns.txt` directly or the generated `rewrite-rules.json`

Docker suitability:
- this repo does not require a Technitium source checkout anymore
- it builds against official DLLs extracted from the Technitium portable archive
- the packaged output is meant to be uploaded/installed into a running Technitium instance, including Docker deployments

Config:
```json
{
  "enable": true,
  "defaultTtl": 300,
  "refreshSeconds": 300,
  "sources": [
    {
      "name": "adguard-dns",
      "enable": true,
      "format": "adguard-filter",
      "url": "https://example.invalid/dns.txt"
    },
    {
      "name": "rewrite-manifest",
      "enable": false,
      "format": "rewrite-rules-json",
      "url": "https://example.invalid/rewrite.json"
    }
  ]
}
```

APP record data template in Technitium UI:
```json
{
  "enable": true,
  "sourceNames": [],
  "overrideTtl": null
}
```

Meaning:
- `enable`: per-record on/off switch
- `sourceNames`: optional array of source names to scope a zone to selected remote sources
- `overrideTtl`: optional per-record TTL override

Rule source:
- this repo publishes `rewrite-rules.json`
- the app fetches that file directly

Operational note:
- create `APP` records on the local zones you want this app to cover
- the app only responds for queries inside the zone passed by Technitium

Build/install layout:
- prepare official SDK DLLs:
```bash
sh scripts/prepare-sdk.sh 14.3.0
```
- package the app:
```bash
sh scripts/package-app.sh
```
- packaged output:
  - `dist/RemoteRewriteApp/`
  - `dist/RemoteRewriteApp.zip`

Docker install model:
- keep Technitium container state on a persistent volume
- build/package this app outside the container
- install the packaged app into Technitium using the app installation flow
- point the app config at either remote `dns.txt`, `rewrite-rules.json`, or both

Suggested APP record usage:
- create `APP` records on zones that should use remote rewrite behavior
- keep upstream routing in native conditional forwarders or `AdvancedForwardingApp`
- keep block/allow behavior in native block list URLs / allowed zones

Lifecycle:
- Technitium re-calls `InitializeAsync()` when app config changes
- the app uses `Dispose()` to clear in-memory state and release synchronization resources
- there is no separate uninstall hook in the Technitium app interface
